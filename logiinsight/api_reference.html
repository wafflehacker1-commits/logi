API Reference

Complete reference for the LogiWebcamClient Python class — the host‑side client for controlling AI models and reading tracking data from the LogiInsight device. 

from logiwebcam import LogiWebcamClient

Requires: pip install paramiko opencv-python numpy

Class Overview

class LogiWebcamClient

Client for controlling LogiInsight AI models over USB NCM (SSH). Manages an SSH connection to the device and provides methods to toggle AI inference models and read real‑time face/hand tracking data.

Supports the context manager protocol (with statement) for automatic connection/disconnection.

Constructor

LogiWebcamClient(ip, username, password, port, timeout)

Initialize the client with connection parameters. Does not open a connection — call connect() or use a with block.

Parameters

Name Type Default Description
ip str "192.168.1.10" Device IP address via USB NCM
username str "makentu" SSH username on the device
password str "MakeNTU@123" SSH password on the device
port int 22 SSH port number
timeout int 10 Connection timeout in seconds 
# Default connection (USB NCM)
cam = LogiWebcamClient()

# Custom IP and timeout
cam = LogiWebcamClient(ip="10.0.0.50", timeout=5)

connect()

cam.connect() → None

Establish an SSH connection to the LogiInsight device. Initializes the USB‑C live stream. If already connected, this is a no‑op.

Raises

LogiWebcamConnectionError — if the SSH connection fails (wrong IP, network unreachable, auth failure). 

cam = LogiWebcamClient()
cam.connect()  # Initializes the USB-C live stream
# ... use the camera ...
cam.disconnect()

disconnect()

cam.disconnect() → None

Close the SSH connection and release the hardware sensor. Safe to call even if not connected. 

# Manual lifecycle
cam.connect()
# ... work with the camera ...
cam.disconnect()  # Releases the hardware sensor

# Preferred: context manager (auto-disconnects)
with LogiWebcamClient() as cam:
    # cam.connect() called automatically
    pass
# cam.disconnect() called automatically

set_ai_config()

cam.set_ai_config(enable_face_detection, enable_face_mesh, enable_hands) → bool

Update the AI model configuration on the device. Writes the configuration to the device's INI file, which the firmware picks up via inotify within ~200 ms.

Constraint: Face Mesh requires enable_face_detection=True. If face detection is disabled, the mesh model has no bounding box input and will produce no landmarks.

Parameters

Name Type Default Description
enable_face_detection bool True Enable face bounding box detection
enable_face_mesh bool False Enable 478‑point face landmark mesh. Requires face detection.
enable_hands bool False Enable 21‑point hand landmark tracking

Returns

True if the configuration was successfully written to the device.

False if the write failed (logged as error).

Raises

LogiWebcamConnectionError — if not connected. 

# Enable face detection + mesh (hands off)
cam.set_ai_config(
    enable_face_detection=True,
    enable_face_mesh=True,
    enable_hands=False,
)

# Enable all models
cam.set_ai_config(
    enable_face_detection=True,
    enable_face_mesh=True,
    enable_hands=True,
)

# Minimum config: face detection only (default)
cam.set_ai_config()

get_latest_ai_data()

cam.get_latest_ai_data() → Optional[Dict]

Polls the NPU for the latest frame data. Reads and parses the JSON output file written by the firmware. Returns None if the file is empty, missing, or malformed.

Important: Always check if results: before accessing the returned data. If the hardware is disconnected, get_latest_ai_data() may return None.

Returns

A dict with keys:

  • "timestamp_ms" — Frame timestamp in milliseconds
  • "faces" — List of detected face objects
  • "hands" — List of detected hand objects

Or None if data is unavailable.

Raises

LogiWebcamConnectionError — if not connected. 

data = cam.get_latest_ai_data()

# Always guard against None
if data:
    ts = data["timestamp_ms"]
    for face in data["faces"]:
        x, y, w, h = face["bbox"]   # [x, y, width, height] in sensor pixels
        conf = face["confidence"]
        lm = face["landmarks"]      # list of [x, y, z] arrays (478 pts if mesh enabled)
        print(f"[{ts}] Face at ({x},{y}) {w}x{h}, conf={conf:.2f}, landmarks={len(lm)}")

    for hand in data["hands"]:
        hx, hy, hw, hh = hand["bbox"]
        hlm = hand["landmarks"]     # list of [x, y, z] arrays (21 pts)
        print(f"[{ts}] Hand at ({hx},{hy}) {hw}x{hh}, landmarks={len(hlm)}")
else:
    print("No data — device may be initializing or disconnected")

is_connected

cam.is_connected → bool property

Check if the SSH connection to the device is active. Returns True if the underlying SSH transport is alive. 

if not cam.is_connected:
    cam.connect()

Core Configuration Table

All options available via set_ai_config() and the underlying INI file:

Option Name Value Range Default
enable_face_detection bool True
enable_face_mesh bool False
enable_hands bool False
max_num_faces Int [1, 5] 1
max_num_hands Int [1, 2] 2
min_detection_confidence Float [0, 1] 0.5

Exceptions

LogiWebcamConnectionError

Raised when the SSH connection to the device fails. This can occur during connect() (wrong IP, unreachable network, auth failure) or when calling methods on a disconnected client.

LogiWebcamDataError

Raised when AI output data cannot be read or parsed. Note: get_latest_ai_data() catches JSON errors internally and returns None instead of raising.
