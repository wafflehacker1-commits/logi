Zero‑Config Setup

Setup Guide

Get your LogiInsight camera streaming AI data to a Raspberry Pi 5 (or any Linux/macOS/Windows host) in under 5 minutes.

Prerequisites

  • ✓ LogiInsight camera with USB‑C cable
  • ✓ Host computer — Raspberry Pi 5, laptop, or desktop running Linux, macOS, or Windows
  • ✓ Python 3.8+ installed on the host
  • ✓ Network access — USB NCM creates a local network interface automatically

3‑Step Initialization

1

Initialize — Connect the Hardware

Plug the LogiInsight camera into your host via USB‑C. The device exposes a USB NCM (Network Control Model) interface, which creates a virtual Ethernet adapter on the host. The device is accessible at 192.168.1.10 by default. 

# Install Python dependencies
pip install paramiko opencv-python numpy

# Verify the device is reachable
ping -c 3 192.168.1.10
2

Enable — Configure AI Models

Use set_ai_config() to choose which AI models to run on the NPU. The configuration is written to an INI file on the device, which the firmware picks up via inotify within ~200 ms.

Important: Face Mesh requires enable_face_detection=True. The mesh model depends on the face detector's bounding box output to locate the face region. 

from logiwebcam import LogiWebcamClient

cam = LogiWebcamClient(ip="192.168.1.10")
cam.connect()

# Enable face detection + face mesh, disable hands
cam.set_ai_config(
    enable_face_detection=True,    # Required for Face Mesh
    enable_face_mesh=True,
    enable_hands=False,
)
3

Capture — Open Camera & Read AI Data

Open the UVC camera with OpenCV (MJPG format), grab a frame, and call get_latest_ai_data() to read the AI output. The firmware writes JSON at ~10 fps.

Tip: Always set the UVC format to MJPG — the AI models require MJPG, not YUV. 

import cv2

# Open UVC camera in MJPG format
cap = cv2.VideoCapture(0, cv2.CAP_V4L2)
cap.set(cv2.CAP_PROP_FOURCC, cv2.VideoWriter_fourcc(*"MJPG"))
cap.set(cv2.CAP_PROP_FRAME_WIDTH, 1920)
cap.set(cv2.CAP_PROP_FRAME_HEIGHT, 1080)

ret, frame = cap.read()              # grab a UVC frame
data = cam.get_latest_ai_data()      # read AI JSON from device

if data:
    for face in data["faces"]:
        x, y, w, h = face["bbox"]    # [x, y, width, height] in sensor pixels
        print(f"Face at ({x}, {y}), size {w}x{h}, conf={face['confidence']}")

cap.release()

Complete Example

The included example.py opens the UVC camera, captures frames, reads AI data, draws face/hand overlays, and saves paired PNG + JSON files: 

# Face detection only (10 frames)
python3 example.py

# Enable 478-point face mesh landmarks
python3 example.py --face-mesh

# Enable 21-point hand tracking
python3 example.py --hands

# Enable all AI models at 1280x720 resolution
python3 example.py --face-mesh --hands --width 1280 --height 720

# Capture 20 frames at 0.5s intervals
python3 example.py --frames 20 --interval 0.5

Output is saved to ./captures/ as paired files: frame_0000.png (annotated image) + frame_0000.json (AI data + metadata).

USB NCM Network Setup

On most systems, the USB NCM interface is configured automatically when you plug in the cable. If you need to set it up manually: 

# Linux: bring up the USB network interface
sudo ip link set usb0 up
sudo ip addr add 192.168.1.1/24 dev usb0

# Verify connectivity
ping -c 1 192.168.1.10

Troubleshooting

Connection refused / timeout

Ensure the USB‑C cable supports data transfer (not charge‑only). Check that the usb0 interface is up with ip link show usb0.

get_latest_ai_data() returns None

The NPU may not have produced a frame yet. Wait 200–500 ms after calling set_ai_config() before polling. Also verify the device hasn't been disconnected.

Face Mesh returns empty landmarks

Face Mesh depends on Face Detection. Ensure enable_face_detection=True is set alongside enable_face_mesh=True.
